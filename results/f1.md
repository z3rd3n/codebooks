# f1.png — reproduction of the paper's Fig. f1 (SE vs SNR, Type I vs Type II)

**What it shows.** Single-polarization ULA study isolating *spatial* compression:
ideal eigen beamforming vs Type II (L=4, OMP + least squares, 3-bit wideband
amplitude, 8-PSK phase) vs Type I (best single DFT beam), for N=4 and N=16
antennas, 1 and 2 streams, on the calibrated sparse multipath channel
(`eval.f1.F1_REPRODUCTION`, 8 paths).

**Why it looks like this.**

* The orderings UB ≥ Type II ≥ Type I hold at every SNR because Type II's
  L=4-beam linear combination can approximate any eigenvector that lives near
  the selected orthogonal group, while Type I is restricted to one grid beam.
* At N=4 the Type II curve is visually on top of the upper bound: 4 beams of
  the orthogonal group span the entire 4-dimensional space, so the only loss
  is amplitude/phase quantization (~0.05 b/s/Hz).
* The gap between Type I and the bound grows from N=4 to N=16: the beam grid
  densifies but a *single* beam captures a shrinking fraction of a multipath
  channel's energy — the paper's central claim for this figure ("as the number
  of antennas increases, the gap becomes more pronounced").
* The 2-stream Type I curve (dotted green) is the farthest from its bound:
  rank-2 Type I must take its second beam from the three i₁,₃ grid offsets
  with a rigid co-phasing structure, which rarely matches the channel's second
  eigenvector ("when the number of data streams increases, the gap is further
  widened").

**Notable / known residual.** The paper's N=4 single-stream Type I curve sits
closer to Type II (~0.15 b/s/Hz) than spec-faithful selection allows (~0.5 here)
on every channel family swept during calibration — documented as a reproduction
residual in the README (erratum 3). All 63 digitized points of the paper figure
are matched within max(±0.6 b/s/Hz, ±8%) by `tests/test_f1_paper_values.py`.

**Config.** `scripts/reproduce_f1.py`; channel/procedure details and the
N_PSK erratum ("3 bits, i.e., N_PSK=4") in `src/nr_csi/eval/f1.py`.
