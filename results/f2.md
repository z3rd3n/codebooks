# f2.png — reproduction of the paper's Fig. f2 (feedback overhead vs L)

**What it shows.** Total feedback bits to cover N₄ = 4 slot intervals at the
paper's configuration ((16,1) ports, rank 2, N₃ = 18, M_v = 5, Q = 2,
N_PSK = 4, K⁽²⁾ = 6, K^NZ = 20), computed from the paper's own Table
bit1/bit2 formulas under the equal-time-coverage convention: R15/R16 have no
temporal compression so they re-report once per interval (×4), R18 covers
all four intervals with one predicted-PMI report.

**Why it looks like this.**

* R15 grows fastest in L because its per-subband i₂ payload is proportional
  to both L and N₃ (every subband re-quantizes 2L−1 amplitudes/phases);
  ×4 re-reporting multiplies all of it.
* R16 replaces the per-subband payload with M_v = 5 delay taps and a
  2L·M_v bitmap, so its bars grow in L but stay ~3× below R15 at L = 4.
* R18's single report is roughly one R16 report plus the Q = 2 Doppler-bin
  duplication of the bitmap — about 1/3 of R16's four-report total and
  ~9× below R15 at L = 4.

**Notable / unexpected.**

* **R16 > R15 at L = 1** (744 vs 488 bits). The paper's own bar chart shows
  R15 above R16 at *every* L, but under the paper's own table formulas R16's
  fixed delay-basis machinery (tap indicators, bitmap, 4+3-bit coefficient
  quantizers over M_v = 5 taps) is not amortized by a single beam, while
  R15's L = 1 payload is tiny ("L = 1 effectively reduces Type II to
  Type I"). The crossover to the familiar ordering happens at L = 2.
* **Absolute bar heights differ from the paper's figure** (paper implies
  R15/R16 ≈ 17× at L = 4; the table formulas give ≈ 2.9× here). This is
  README erratum 4: the paper's bar values are not derivable from its own
  Tables bit1/bit2 with the stated parameters under any consistent
  accounting we found. The qualitative claims that *are* derivable (R15 ≫
  R16 > R18 for L ≥ 2, same growth rate in L, gap growing with N₃/N₄) hold
  and are asserted in `tests/test_overhead.py`.
