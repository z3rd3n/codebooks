# Chapter 3 ÔÇö R15 Type II codebook (regular + port selection)

* **Spec:** TS 38.214 **┬º5.2.2.2.3 "Type II Codebook"** (regular) and
  **┬º5.2.2.2.4 "Type II Port Selection Codebook"** (port selection).
  *(Earlier drafts of this chapter swapped the two clause numbers ÔÇö the correct
  mapping is `.3 = Type II regular`, `.4 = Type II Port Selection`.)*
* **Code:** [type2_r15.py](../../src/nr_csi/codebooks/type2_r15.py) ÔÇö classes
  `R15Type2Codebook`, `TypeIIRestriction`
* **Ranks:** 1ÔÇô2 (`The UE shall not report RI > 2`, ┬º5.2.2.2.3 / ┬º5.2.2.2.4).
* **Antenna ports:** 4, 8, 12, 16, 24 or 32 CSI-RS ports
  ($P_\text{CSI-RS} \in \{4,8,12,16,24,32\}$), with $(N_1,N_2)$ /
  $(O_1,O_2)=(4,4)$ from Table 5.2.2.2.1-2 (regular) or any $P$ (port selection).
* **Compression:** spatial only ÔÇö a linear combination of $L$ DFT beams, with an
  independent complex coefficient **per beam, per polarization, per subband**.
* **Prereq:** [Chapter 0 ÔÇö Foundations](00-foundations.md) (DFT bases,
  Algorithm 1, quantization tables, the shared UE algorithm).

Type II is a different philosophy from Type I. Instead of "pick the best beam,"
the UE reports "here is a weighted sum of $L$ beams that approximates the
eigenvector." The weights (amplitudes + phases) are quantized and sent **for every
subband**, so the report is far larger but tracks the channel much more closely.
This is the structural template that R16/R17/R18 compress further.

> **Why two clauses.** ┬º5.2.2.2.3 builds the spatial basis from *oversampled DFT
> beams* (the gNB knows the array geometry, the channel is in the angular
> domain). ┬º5.2.2.2.4 swaps the DFT beams for *standard-basis (single-port)
> vectors* ÔÇö used when the CSI-RS is already beamformed / port-reduced (e.g.
> reciprocity-based port selection), so the UE only has to pick which of the
> already-formed ports to keep. **Everything between the spatial basis and the
> final normalization is shared** between the two, which is why one class
> implements both via the `port_selection` flag.

## 1. The reconstruction formula

For layer $l$ and frequency unit (subband) $t$, the per-polarization precoder is a
beam combination. The spec writes the layer-$l$ column of $W$ (Table 5.2.2.2.3-5
for regular, Table 5.2.2.2.4-1 for port selection; paper Table `tabtypeii`) as

$$w^{(l)}_t = \frac{1}{\sqrt{N_1 N_2 \sum_{i=0}^{2L-1} \big(p^{(1)}_{l,i}\, p^{(2)}_{l,i}\big)^2}}
\begin{bmatrix}
\displaystyle\sum_{i=0}^{L-1} v_{m_1^{(i)},m_2^{(i)}}\; p^{(1)}_{l,i}\, p^{(2)}_{l,i}\, \varphi_{l,i} \\[10pt]
\displaystyle\sum_{i=0}^{L-1} v_{m_1^{(i)},m_2^{(i)}}\; p^{(1)}_{l,i+L}\, p^{(2)}_{l,i+L}\, \varphi_{l,i+L}
\end{bmatrix}.$$

* $v_{m_1,m_2}$ ÔÇö the selected oversampled DFT beam (same $L$ beams for both
  polarizations); $m_1 = O_1 n_1 + q_1$, $m_2 = O_2 n_2 + q_2$. **Port-selection
  variant:** $v$ is replaced by the standard-basis (single-port) vector ÔÇö see ┬º4.
* $p^{(1)}_{l,i}$ ÔÇö **wideband** amplitude, one per beam┬Àpol, from the 3-bit
  index $k^{(1)}_{l,i}$ via Table 5.2.2.2.3-2 (`R15_WB_AMP`).
* $p^{(2)}_{l,i}$ ÔÇö **subband** amplitude, from the 1-bit index $k^{(2)}_{l,i}$
  via Table 5.2.2.2.3-3 (`R15_SB_AMP`); $=1$ when `subbandAmplitude=false`.
* $\varphi_{l,i} = e^{\,j 2\pi c_{l,i}/N_\text{PSK}}$ ÔÇö PSK phase, with the
  per-coefficient $N_\text{PSK} \in \{4, 8\}$ described in ┬º5.
* The denominator normalizes each layer column to unit norm; the leading
  $N_1 N_2$ is the beam-energy factor (**dropped** in the port-selection variant,
  Table 5.2.2.2.4-1, where the spec quantity is $\sqrt{\sum_i (p^{(1)}p^{(2)})^2}$
  with no $N_1 N_2$).

There are $2L$ coefficients per layer (the index $i$ runs $0..2L{-}1$: beams
$0..L{-}1$ on polarization 0, $L..2L{-}1$ on polarization 1). For rank 2 the two
layers $W = \tfrac{1}{\sqrt{\upsilon}}[\,w^{(0)}\ \ w^{(1)}\,]$ share the same
spatial basis but carry independent coefficients; the $1/\sqrt{\upsilon}$ factor
(spec: the codebook in Table 5.2.2.2.3-5 normalizes by the number of layers) is
applied last.

## 2. Configuration

```python
R15Type2Codebook(
    antenna,
    N3 = 1,
    L = 4,                    # numberOfBeams in {2, 3, 4}
    n_psk = 8,                # phaseAlphabetSize 4 or 8
    subband_amplitude = False,# subbandAmplitude on/off
    port_selection = False,   # regular vs port-selection variant
    d = 1,                    # portSelectionSamplingSize (PS only), 1..4, d<=min(P/2,L)
    restriction = None,       # TypeIIRestriction (regular only)
)
```

* **`L`** ÔÇö how many beams are combined ($L\in\{2,3,4\}$). More beams = better
  fit, more bits. The code rejects `L` outside $\{2,3,4\}$ and any `L > N1*N2`
  (regular) / `L > P/2` (a group cannot supply more than its beam count).
* **`n_psk`** ÔÇö phase resolution: 8-PSK = 3 bits, QPSK = 2 bits
  (`phaseAlphabetSize`).
* **`subband_amplitude`** ÔÇö whether a 1-bit per-subband amplitude refinement is
  added on top of the wideband amplitude.
* **`port_selection`** ÔÇö the **┬º5.2.2.2.4** variant: beams are replaced by
  *standard basis vectors* (individual ports), useful when the channel is already
  in the beam/antenna domain (e.g. reciprocity-based port reduction). `d` is then
  the `portSelectionSamplingSize`, constrained to $1\le d\le\min(P/2,L)$ in code
  (the spec range is $d\in\{1,2,3,4\}$, $d\le P/2$). Subset restriction does not
  apply to the PS variant and the code rejects passing both.

> ÔÜá´©Å The codebase clamps `d` to `d <= min(P/2, L)`. TS 38.214 ┬º5.2.2.2.4 only
> requires $d\in\{1,2,3,4\}$ and $d\le P/2$ (no `d <= L` requirement). The extra
> `d <= L` bound is a codebase convenience, not a spec rule. **[partial ÔÇö stricter
> than spec]**

## 3. The PMI

```python
@dataclass
class R15Type2PMI:
    rank: int
    # regular variant ÔÇö beam selection:
    q1, q2: int | None       # orthogonal group (q1,q2)
    i12:    int | None       # L-beam combination (Algorithm 1)
    # port-selection variant:
    i11_ps: int | None       # initial port index
    # per-layer coefficient fields:
    i13: list[int]           # strongest-coefficient index per layer, in 0..2L-1
    k1:  np.ndarray          # (v, 2L) wideband amplitude indices  -> p^(1)
    k2:  np.ndarray          # (v, N3, 2L) subband amplitude indices -> p^(2)
    c:   np.ndarray          # (v, N3, 2L) phase indices -> phi
```

Mapping to spec PMI elements: $(q_1,q_2)$ is reported as the rotation indicator
$i_{1,1}$, the beam combination is $i_{1,2}$, the strongest coefficient is
$i_{1,3,l}$, the wideband amplitudes are $i_{1,4,l}$, the phases are $i_{2,1,l}$,
and the subband amplitudes are $i_{2,2,l}$.

## 4. Spatial basis (`_basis`)

* **Regular:** decode $i_{1,2}$ to $L$ beam indices $(n_1^{(i)}, n_2^{(i)})$ via
  Algorithm 1, map into oversampled grid coordinates inside group $(q_1,q_2)$
  ($m_1 = O_1 n_1 + q_1$, $m_2 = O_2 n_2 + q_2$), and stack the DFT beams ÔåÆ shape
  $(L, P/2)$.
* **Port selection:** $B[i] = e_{(i_{1,1}\cdot d + i)\bmod (P/2)}$ ÔÇö the standard
  basis vector for a single port, with stride $d$. No DFT, no $N_1 N_2$ factor.

The `_beta_factor()` is $N_1 N_2$ (regular) or $1$ (PS) ÔÇö the normalization
difference between the two variants.

---

## 5. Coefficient partition and reporting reduction

Type II saves bits by sending **fewer, coarser** coefficients for the weaker
beams. `_partition(k1_row, i_star)` splits the $2L$ coefficients of a layer into:

* **strong** ÔÇö the $\min(M_l, K^{(2)})$ largest by wideband amplitude, excluding
  the strongest; these get full PSK phase *and* (if enabled) a subband amplitude.
* **weak** ÔÇö the remaining nonzero coefficients; phase falls back to **QPSK**
  regardless of `n_psk`, and no subband amplitude.
* **zero** ÔÇö coefficients with wideband amplitude index 0 ($p^{(1)}=0$); not
  reported at all (no phase, no amplitude).

Here $M_l$ = number of nonzero wideband amplitudes, and the cap is

$$K^{(2)} = \begin{cases} 4 & L \in \{2,3\} \\ 6 & L = 4 \end{cases}$$

(`k2_cap`). `_phase_alphabets` returns the per-coefficient PSK size used by the
phase field. The strongest coefficient itself is always first in the ranking and
carries no phase/amplitude bits (see [Foundations ┬º11](00-foundations.md#11-strongest-coefficient-and-reporting-reduction-conventions)).

## 6. Reconstruction in code (`precoder`)

For each layer $l$ and subband $t$:

```python
p1   = R15_WB_AMP[k1[l]]                       # (2L,) wideband amplitudes
p2   = R15_SB_AMP[k2[l,t]] if sa else 1        # (2L,) subband amplitudes
phi  = exp(2j*pi * c[l,t] / sizes)             # per-coeff PSK (weak -> QPSK)
amp  = p1 * p2
coeff= amp * phi
beta = beta_factor * sum(amp**2)               # normalization
w    = [ B.T @ coeff[:L] ; B.T @ coeff[L:] ]   # stack the two polarizations
W[0,t,:,l] = w / sqrt(beta)
```

then the whole matrix is scaled by $1/\sqrt{\text{rank}}$. This is exactly the
formula in ┬º1, with `sizes` implementing the strong/weak QPSK fallback.

## 7. Selection (`select`)

The UE side follows the [shared algorithm](00-foundations.md#9-the-shared-ue-side-algorithm):

1. **Targets.** $H_t = H[-1]$; compute the rank-$v$ eigen targets (scaled to
   unit columns).
2. **Spatial basis.**
   * Regular: `select_group_and_beams` ÔÇö for each orthogonal group, project the
     targets onto its beams, keep the group whose top-$L$ beams capture the most
     energy, and encode those beams (Algorithm 1). A subset restriction passes an
     `allowed(q1,q2)` mask so prohibited beams are skipped.
   * PS: `select_ps_initial` ÔÇö choose the starting port $i_{1,1}$ whose $L$
     consecutive (stride-$d$) ports carry the most energy.
3. **Coefficients.** `ls_coefficients(B, targets, beta_factor)` ÔåÆ complex
   coefficients $(v, N_3, 2L)$.
4. **Quantize** (`_quantize_into`):
   * Pick the strongest coefficient $i^\star$ per layer by RMS-over-subbands
     amplitude (constrained to an *uncapped* beam when a subset restriction is
     active, so the unreported $k^{(1)}{=}7$ stays honest).
   * Rotate every subband so $i^\star$ is real (zero phase).
   * Wideband amplitude $k^{(1)}$ = nearest level of $|{\cdot}|/|i^\star|$ clipped
     to 1; set $k^{(1)}[i^\star]=7$; apply restriction caps if any.
   * Per subband: quantize phases for non-zero, non-strongest coefficients at the
     partition-dependent PSK size; if `subband_amplitude`, quantize 1-bit subband
     amplitudes for the **strong** set only.

## 8. Codebook subset restriction (`TypeIIRestriction`)

The ┬º5.2.2.2 *codebookSubsetRestriction* limits which beams may be used and caps
their amplitudes (paper eqs. a58/a59 + Table `tabmaxap`):

```python
@dataclass
class TypeIIRestriction:
    beta1: int           # Algorithm 2 index selecting 4 restricted groups of O1*O2
    b2:    np.ndarray    # (4, N1*N2) 2-bit codepoints {0,1,2,3} per beam
```

* `beta1` chooses the 4 restricted orthogonal-vector groups (Algorithm 2).
* For each restricted group, `b2` gives a 2-bit max-amplitude codepoint per beam.
  `MAX_K1 = {0:-1, 1:5, 2:6, 3:7}` maps codepoint ÔåÆ the largest admissible
  wideband-amplitude index ($-1$ = beam prohibited).
* `k1_caps(antenna, q1, q2)` returns the per-beam $k^{(1)}$ ceiling for a group;
  unrestricted groups return 7 (no cap). The selector uses it to skip prohibited
  beams and clip amplitudes; the strongest coefficient is forced onto an uncapped
  beam.

## 9. Overhead (`overhead_bits`)

```python
# spatial:
regular:  i11 = ceil(log2(O1*O2)),  i12 = ceil(log2(C(N1*N2, L)))
PS:       i11 = ceil(log2(ceil(P / (2*d))))
# per layer:
i13 = rank * ceil(log2(2L))           # strongest-coefficient index
i14 = rank * 3 * (2L - 1)             # wideband amplitudes (strongest skipped)
# per subband, per layer (accumulated):
i21 += N3 * (phases)                  # see below
i22 += N3 * len(strong)               # subband amplitudes (only if sa)
```

The per-subband phase term depends on the partition:

* `subband_amplitude` **on**: `len(strong)*log2(n_psk) + 2*len(weak)` per subband
  (strong at full PSK, weak at QPSK; strongest skipped).
* `subband_amplitude` **off**: `(Ml - 1) * log2(n_psk)` ÔÇö all nonzero (except the
  strongest) at full PSK.

The dominant cost is $i_{2,1}$ (phases) because it is **per subband per beam**.
This is precisely the redundancy R16 attacks: across subbands the phases vary
smoothly (they are the frequency response of a few delays), so R16 replaces the
$N_3$-wide $i_{2,1}$ block with a handful of **delay-tap** coefficients
([Chapter 4](04-etype2-r16.md)).

**Worked dimensions** ÔÇö $N_1{=}4,N_2{=}2$ ($P{=}16$), $L{=}4$, $N_3{=}10$,
8-PSK, rank 1, no subband amplitude, all $2L{=}8$ beams nonzero:
$i_{1,1}=\lceil\log_2 16\rceil=4$, $i_{1,2}=\lceil\log_2\binom{8}{4}\rceil=\lceil\log_2 70\rceil=7$,
$i_{1,3}=\lceil\log_2 8\rceil=3$, $i_{1,4}=3\cdot7=21$,
$i_{2,1}=10\cdot(8{-}1)\cdot3=210$. Total Ôëê **245 bits** ÔÇö an order of magnitude
more than Type I's 27, and almost all of it is the per-subband phase block.

## 10. Port-selection variant in one line

Set `port_selection=True`: `_basis` becomes stride-$d$ standard basis vectors,
`_beta_factor` becomes 1, the spatial report becomes a single initial-port index
$i_{1,1}$, and everything else (coefficients, reporting reduction, normalization
structure) is unchanged. It assumes the channel is already concentrated on a few
ports (beam-domain CSI-RS), so it skips the DFT beam search.
